# V1: GF(2) 위 zkEVM — Multivariate ZK-STARK + Proof Tree VM

## 확정 스펙

| 항목 | 결정 |
|---|---|
| GPU 프레임워크 | WebGPU (`wgpu = "28"`) |
| 필드 | GF(2^128) 이진 타워 확장체 |
| 다항식 커밋 | Batch Tensor PCS (모든 재귀 레벨 단일 배치 커밋) |
| 목표 규모 | ~2^20 AND gates / shard |
| ZK 증명 구조 | Sumcheck + GKR + 가변 깊이 재귀 (`RecursiveConfig`) |
| 해시 (외부) | Blake3 (Fiat-Shamir 최상위, Merkle) |
| 해시 (인서킷) | Vision-4 (재귀 검증자 회로 내 Fiat-Shamir) |
| ZK 블라인딩 | 랜덤 블라인딩 다항식 추가 |
| 비대화형 | Fiat-Shamir 변환 (완전 비대화형) |
| 회로 방식 | Register-based Meta-VM 트레이스 → Boolean 회로 행(Row) |
| 비싼 연산 | Advice Witness 패턴 (외부 계산, 회로에서 검증만) |
| 증명 구조 | 형식적 Proof Tree (타입 규칙 기반 EVM 전이 증명) |
| 256-bit 산술 | 32-bit Micro-Op 분해 (XOR/ROT = 0 AND, ADD32 = 32 AND) |

---

## 전체 설계 철학

```
기존 zkEVM:  EVM 실행 → 산술 회로 (모든 것이 회로 안)
이 설계:     EVM 실행 → [Proof Tree + Advice Tape]
                                  │                    │
                         타입 체크 (무료)     Advice 검증 회로 (최소 게이트)
                                  └──────────────────┘
                                    ZK-STARK (GPU, 재귀)
```

**세 층의 분리:**
1. **Proof Tree 레이어** — EVM 전이를 타입 규칙으로 검증 (구조적, 비ZK)
2. **Meta-VM 레이어** — Proof Tree를 구축하는 레지스터 머신, 트레이스가 ZK 증인
3. **ZK 레이어** — Advice 검증 제약 + GKR-Sumcheck + 재귀 집계 + Batch Tensor PCS

## Layer 1: Proof Tree (crates/evm-types)

EVM 실행을 **타입 유도 트리(derivation tree)** 로 표현한다.
타입 체커는 트리 구조만 검증하고, 산술 정확성은 ZK에 위임한다.

### EVM 타입 규칙

각 opcode는 스택·메모리 상태를 변환하는 타입 규칙으로 인코딩된다:

```
ADD  :  Stack(a:U256, b:U256, rest) → Stack((a+b) mod 2^256, rest)
MUL  :  Stack(a:U256, b:U256, rest) → Stack(a*b mod 2^256,   rest)
DIV  :  Stack(a:U256, b:U256, rest) → Stack(a/b,             rest)   [b≠0]
JUMP :  Stack(dst:U256, rest)  ×  [dst ∈ JUMPDEST_TABLE]  →  PC=dst
MLOAD:  Stack(addr:U256, rest) × Mem  →  Stack(val:U256, rest) × Mem
KECCAK256: Stack(off, len, rest) × Mem  →  Stack(hash, rest) × Mem
```

### Proof Tree 노드

```rust
// crates/evm-types/src/proof_tree.rs
pub enum ProofNode {
    // 리프: 단일 opcode 전이 + ZK 산술 증명
    Leaf {
        opcode:     Opcode,
        pre_state:  EvmState,
        post_state: EvmState,
        leaf_proof: LeafProof,   // 산술 정확성 ZK 서브증명
    },
    // 순차 합성: left.post == right.pre 이면 합성 가능
    Seq {
        left:  Box<ProofNode>,
        right: Box<ProofNode>,
    },
    // 조건 분기: JUMPI
    Branch {
        cond:      Box<ProofNode>,
        taken:     Box<ProofNode>,
        not_taken: Box<ProofNode>,
    },
}

// 타입 체커: O(n) 구조 검증만, ZK 없음
pub fn type_check(node: &ProofNode) -> Result<EvmStateType, TypeError>;
```

### GPU 워크그룹 = 서브트리 경계

```
EVM 실행 (1024 opcodes)
  ├── 서브트리 A (256 opcodes) → workgroup 0
  ├── 서브트리 B (256 opcodes) → workgroup 1
  ├── 서브트리 C (256 opcodes) → workgroup 2
  └── 서브트리 D (256 opcodes) → workgroup 3
           ↑ Compose 노드 = 재귀 집계 경계 (자연스러운 분할)
```

## Layer 2: Meta-VM (crates/vm)

Proof Tree를 구축하는 **레지스터 기반 소형 VM**.
VM 실행 트레이스 = ZK 증인.

### Micro-Op ISA

```rust
pub type Reg = u8;       // 32개 레지스터 (0..31)
pub type CarryReg = u8;  // 8개 캐리 플래그

pub enum MicroOp {
    // ── 32-bit 산술 ──────────────────────────────────────
    /// 32-bit 캐리 가산기.  비용: 32 AND (캐리 체인)
    Add32 { dst: Reg, a: Reg, b: Reg, cin: CarryReg, cout: CarryReg },
    /// 32×32→64 곱셈.       비용: 32 AND (Karatsuba)
    Mul32 { dst_lo: Reg, dst_hi: Reg, a: Reg, b: Reg },

    // ── Bitwise (비용: 0 AND — GF(2) 덧셈 또는 와이어) ──
    Xor32 { dst: Reg, a: Reg, b: Reg },        // 0 AND
    And32 { dst: Reg, a: Reg, b: Reg },        // 32 AND
    Not32 { dst: Reg, a: Reg },                // 0 AND
    Rot32 { dst: Reg, a: Reg, amt: u8 },       // 0 AND (와이어 치환)
    Shr32 { dst: Reg, a: Reg, cin: Reg, amt: u8 }, // 0 AND

    // ── Keccak Chi (비선형 스텝 전용) ───────────────────
    /// dst = a XOR ((NOT b) AND c).  비용: 32 AND
    Chi32 { dst: Reg, a: Reg, b: Reg, c: Reg },

    // ── Advice 로드 + 검증 ───────────────────────────────
    /// 외부 오라클 값 주입.  이후 Check* 명령으로 반드시 검증.
    Advice { dst: Reg, oracle_idx: u32 },
    /// a == q*b + r  AND  r < b  (나눗셈 검증).  비용: ~4K AND
    CheckDiv  { q: Reg, r: Reg, a: Reg, b: Reg },
    /// q == a*b mod 2^256  (곱셈 검증).           비용: ~4K AND
    CheckMul  { q_lo: Reg, q_hi: Reg, a: Reg, b: Reg },
    /// a * a_inv == 1  (역원 검증).                비용: ~4K AND
    CheckInv  { a: Reg, a_inv: Reg },
    /// 0 <= r < 2^bits  (범위 체크).               비용: ~bits AND
    RangeCheck { r: Reg, bits: u32 },

    // ── 증명 트리 합성 ───────────────────────────────────
    /// ProofNode::Seq(left ; right).  비용: ~10 AND (타입 일치)
    Compose   { dst: Reg, left: Reg, right: Reg },
    /// opcode 타입 규칙 체크.         비용: ~50 AND (룩업)
    TypeCheck { opcode: u8, pre: Reg, post: Reg },

    // ── Keccak: 리프 ZK 서브증명 ────────────────────────
    /// 이 스텝의 Keccak을 별도 ZK 서브증명으로 처리.
    /// 회로는 서브증명 커밋만 흡수.
    KeccakLeaf { dst_commit: Reg, input: Reg },

    // ── 메모리 / 증인 로드 ───────────────────────────────
    Load  { dst: Reg, addr: u32 },
    Store { addr: u32, src: Reg },
    Done,
}
```

### EVM Opcode → Micro-Op 컴파일 예시

**`ADD` (U256, 256 AND gates, 8 rows):**
```
Add32 {dst:r8,  a:r0, b:r4, cin:c0, cout:c1}   ; limb 0
Add32 {dst:r9,  a:r1, b:r5, cin:c1, cout:c2}   ; limb 1
...
Add32 {dst:r15, a:r7, b:r11,cin:c7, cout:c0}   ; limb 7
```

**`DIV a b` (Advice, ~4K AND, 3 rows):**
```
Advice    {dst:q, oracle_idx:0}              ; q = a/b  외부 계산
Advice    {dst:r, oracle_idx:1}              ; r = a%b  외부 계산
CheckDiv  {q, r, a, b}                       ; a==q*b+r AND r<b  (~4K AND)
; 기존 monolithic 방식 ~200K AND → 50배 감소
```

**`MULMOD a b N` (~8K AND, 5 rows):**
```
Advice    {dst:q,  oracle_idx:0}             ; q = (a*b)/N
Advice    {dst:r,  oracle_idx:1}             ; r = (a*b)%N  → 결과
CheckMul  {q_lo:t0, q_hi:t1, a, b}          ; t = a*b       (~4K AND)
CheckMul  {q_lo:t2, q_hi:t3, q, N}          ; s = q*N       (~4K AND)
; t == s + r  →  XOR 체크 (free)
```

**`KECCAK256` (리프 ZK, 메인 회로 0 AND):**
```
KeccakLeaf {dst_commit:r0, input:r1}         ; 서브증명 커밋 흡수만
; Keccak 38K AND는 별도 리프 ZK 서브회로에서 처리
```

### 게이트 예산 비교 (XOR=0 AND 기준)

| 연산 | 기존 AND | Advice/Micro-Op | 감소 배율 |
|---|---|---|---|
| U256 ADD | 256 | 256 | 1× |
| U256 MUL | ~50K | **4K** (Mul32 × 128) | 12× |
| U256 DIV/MOD | ~200K | **4K** (CheckDiv) | 50× |
| U256 MULMOD | ~400K | **8K** (CheckMul × 2) | 50× |
| Modular INV | ~500K | **4K** (CheckInv) | 125× |
| Keccak-256 θρπ | ~100K | **0** (XOR/ROT = free) | ∞ |
| Keccak-256 Chi | ~150K | **38K** (Chi32 × 24 rnd) | 4× |
| TypeCheck | — | **50** (룩업) | — |
| Compose | — | **10** | — |

## Layer 3: ZK 증명 (crates/sumcheck, pcs, shard, recursive, stark)

### 3-1. GF(2^128) 필드 (crates/field)

**이진 타워 구조:**
```
GF(2^128) = GF(2^64)[x] / (x² + x + α₆₄)
GF(2^64)  = GF(2^32)[x] / (x² + x + α₃₂)
...  (7단계 타워)
GF(2^2)   = GF(2)[x]    / (x² + x + 1)
```

**Rust 타입:**
```rust
#[repr(C)]
#[derive(Copy, Clone, Pod, Zeroable)]
pub struct Gf2_128([u32; 4]);

impl Gf2_128 {
    pub fn add(self, rhs: Self) -> Self;   // XOR
    pub fn mul(self, rhs: Self) -> Self;   // Karatsuba 8단계 타워
    pub fn inv(self) -> Self;              // Fermat: self^(2^128-2)
}
```

**WGSL (`shaders/gf2_128.wgsl`):**

### 3-6. 가변 깊이 재귀 (crates/recursive)

```rust
pub struct RecursiveConfig {
    pub total_vars:  u32,   // 전체 변수 수 (기본: 20)
    pub shard_vars:  u32,   // 샤드당 변수 수 (기본: 8)
    pub fan_in:      u32,   // 집계 fan-in k  (기본: 8)
}

impl RecursiveConfig {
    pub fn n_shards(&self) -> u32 { 1 << (self.total_vars - self.shard_vars) }
    pub fn depth(&self) -> u32 { /* ceil(log_{fan_in}(n_shards)) */ }
    pub fn instances_at_level(&self, l: u32) -> u32;
}
```

| L2 recursive | 8 | ~500K |

**재귀 증명 루프:**
```rust
pub fn prove_recursive(
    shard_proofs:  Vec<ShardProof>,
    config:        &RecursiveConfig,
    batch_pcs:     &mut BatchTensorPCS,
    gpu:           &GpuContext,
    transcript:    &mut Blake3Transcript,
) -> RootProof {
    let mut current = shard_proofs;
    for level in 0..config.depth() {
        let circuit = RecursiveCircuit { level, config: config.clone() }.build();
        current = gpu_prove_instances(&circuit, current, gpu, transcript);
        batch_pcs.add_level_witnesses(level, &current);
        transcript.absorb("level_claims", &extract_claims(&current));
    }
    prove_root(current, batch_pcs, transcript, gpu)
}
```

### 3-7. Batch Tensor PCS (crates/pcs)

**커밋:**
```
모든 레벨 증인 → 단일 행렬
  rows: 2^(n/2)   cols: 2^(n/2)
  각 행에 랜덤 선형 코드 G 적용 (rate 1/4, 100-bit security)
  Merkle 트리 (Blake3) → 32B root 커밋
```

**개방 (r = r_lo ∥ r_hi):**
```
1. t_col = Σ_i eq(r_hi, i) * row_i   (GPU: batch_row_rlc_kernel)
2. encode(t_col) == Σ_i eq(r_hi, i) * encode(row_i)  (선형 일관성)
3. t_col(r_lo) = y  (내적 Sumcheck)
4. Merkle 경로로 쿼리된 열 인증
```

## 전체 증명 파이프라인

```
입력: EVM bytecode + 실행 입력
    │
    ▼ [stark::Prover::prove]
    │
    ├─ 1. Meta-VM 실행 (vm::exec + oracle::ComputingOracle)
    │       EVM opcode → MicroOp 컴파일 → 실행 → MachineTrace
    │       부산물: advice_tape: Vec<U256>
    │
    ├─ 2. Proof Tree 타입 체크 (evm-types::type_check)
    │       ProofTree 구조 검증 — O(n), 비ZK, 실패 시 증명 중단
    │
    ├─ 3. 트레이스 → Boolean 회로 행 (circuit)
    │       MachineTrace Row → 다중선형 확장 테이블 (MLE)
    │       advice_tape → advice 컬럼 MLE
    │       Check* 명령 → 다항식 제약 (GKR / Sumcheck)
    │
    ├─ 4. Batch Tensor PCS 증인 커밋 (pcs)
    │       전체 증인 MLE → 단일 32B Merkle root
    │
    ├─ 5. 서브트리 경계 기반 샤딩 (shard)
    │       GPU 4096 workgroup 병렬 Sumcheck
    │       Blake3 fork 도메인 분리 → ShardProof × 4096
    │
    ├─ 6. 가변 깊이 재귀 집계 (recursive)
    │       L0..L3 재귀 (RecursiveCircuit + Vision-4 Fiat-Shamir)
    │       각 레벨 증인 → batch_pcs에 추가
    │
    ├─ 7. Batch Tensor PCS 개방 (pcs)
    │       루트 클레임 평가 지점에서 배치 개방 (~80KB)
    │
    └─ 8. Proof 직렬화 (bincode)
           { advice_commit, type_cert, recursive_proof, batch_open }
```

## 검증자 체크리스트

1. `batch_commit` absorb → `query_cols` Fiat-Shamir 재현
2. `type_cert` 타입 체크 (구조 검증, O(n), 비ZK)
3. Root Sumcheck 라운드 다항식 일관성
4. L3→Root 재귀 회로 Vision-4 챌린지 재현 + 검증
5. Check* 제약 (Advice 일관성) 확인
6. Batch PCS 개방: `t_col` 선형 일관성 + Merkle 경로 + 내적 Sumcheck
7. 최종 클레임 = 회로 출력 MLE(r) 일치

## ZK 블라인딩

```
각 레벨 l의 증인 MLE W_l(x)에 대해:
  W_l_blinded(x) = W_l(x) ⊕ r_l * B_l(x)

  r_l: Gf2_128::random()  (zeroize on drop)
  B_l: 랜덤 다중선형 다항식, {0,1}^n 위 평가 = 0
     → 합산 클레임 불변, 증인 값 완전 은폐

advice_tape도 각 항목에 독립 블라인딩 적용.
```
